# Contributing

Thanks for helping keep this list accurate — that's the entire value of it.

## Adding or updating a provider

1. Link directly to the provider's **own official** pricing or rate-limit docs. Third-party summaries (including other curated lists) are a fine starting point for research, but the link in the table should point to the primary source.
2. Include the date you personally checked the numbers. We use that as the `Last verified` date.
3. Note the fine print: phone verification requirements, "evaluation only / not for production" clauses, data-training opt-ins, resale/commercial-use restrictions, anything that would surprise someone relying on the free tier.
4. If you're not 100% sure a number is current, mark the row with ⚠️ and say so in your PR description rather than presenting it with false confidence — an honestly-flagged uncertain entry is more useful than a confident but wrong one.

## Reporting a broken link or outdated entry

Open an issue — or if the weekly link-check workflow already opened a "🔗 Broken link report" issue, feel free to just comment there with what you found.

## Style

- Keep the tables scannable: short phrases, not paragraphs, in each cell.
- Prefer concrete numbers (req/min, tokens/day) over vague marketing language like "generous limits."
- If a provider doesn't publish exact numbers publicly, say so explicitly rather than inventing a plausible-looking figure.
